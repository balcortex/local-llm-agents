# Codebase Exploration Skill

Use this skill when you need to understand a repository before editing or reviewing it.

## Procedure

1. Inspect the root directory.
2. Read README and documentation files.
3. Identify dependency files and package managers.
4. Locate entry points.
5. Locate tests.
6. Identify configuration files.
7. Trace the requested feature or workflow.
8. Summarize only what is supported by repository evidence.

## Output

```markdown
## Summary
<short project summary>

## Important files
- `<path>`: <why it matters>

## Main flow
1. <step>
2. <step>

## Risks or questions
- <item>
```

## Constraints

- Do not modify files.
- Do not run destructive commands.
- Do not invent missing architecture.
